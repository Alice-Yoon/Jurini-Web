## 주린이 달력 (웹)

기획자, 디자이너, 개발자 총 8명이 한 팀이 되어
미국 주식 배당금/배당락일/배당지급일을 한눈에 볼 수 있는 달력 앱과 웹 제작을 하였다.

그 중 나는 웹 개발의 Front-End를 맡아서 진행했고,
달력 자체의 UI를 제외한 모든 기능을 맡아 개발했다.

웹은 사용자에게 제일 베이직한 정보만 전달함으로서 주린이 달력 서비스의 '맛보기'를 제공하고,
앱으로 유도하는 징검다리 역할을 한다.

### 프로젝트 기한
2020.07 ~ 2020.10 (총 3개월 반)

### 기술 스택
React.js + Redux

### 개발 단계의 어려움과 해결

- **달력에서 클릭된 날짜는 분홍색 스타일링을 적용해서 나타내줘야 하는데 처음에는 어떻게 구현해야 하는지 감이 잡히지 않았다**

<img width="111" alt="스크린샷 2021-02-05 오전 3 23 11" src="https://user-images.githubusercontent.com/53468065/106937744-7b1e3d00-6761-11eb-91d6-b7a3e8adcbfd.png">


: 선택된 날짜를 milliseconds로 변경하여 전역 state로 관리하고, 달력 날짜 칸 컴포넌트에서 자신의 날짜의 milliseconds와 비교하여 같을 때만 분홍색 스타일링을 적용하였다.

- **개발 기한에 맞추느라 크롤링 해온 데이터를 사용할때, back-end에서 api를 프론트의 필요에 맞추어 가공할 수 없었다.**
예를들어, 달력의 각 날짜에 해당하는 데이터를 바로 가져올 수 있는 api가 없어서 월별로 가지고 와야 했고,
종목 검색시에도 검색어로 한번에 가져올 수 없고 두개의 api를 거쳐야 했다.

: 프론트에서 필요한 api들을 직접 가공해서 적용했다.

-----

혼자 프론트엔드 독학한지 4개월 쯔음 지났을 때 처음 참가한 그룹 프로젝트였다. 앱과 웹을 나누어 개발하였고, 각각 2명의 개발 팀원들이 맡게 되었다.

그런데 웹 프론트엔드의 인력이 부족해 팀 내 유일한 취준생이었던 내가 웹 프론트엔드를 책임지게 되었다. 부담감과 함께 책임감을 강하게 느꼈으며, 조금 두렵기도 했지만 용기내어 맡아서 진행하게 되었다.

개발인력 뿐 아니라, 기획자와 디자이너도 함께 팀을 이루어 진행하였기 때문에 웹개발의 전체적인 협력 프로세스를 얕게나마 경험해 볼 수 있었다.

데이터 크롤링 한계의 문제로 실제 상용화까지 하진 못하여서 원래 예정되어 있던 크롬 익스텐션 개발까지는 하지 못하여 아쉬움도 있었다.

하지만 총 5팀이 MVP를 두고 경쟁하는 프로젝트였는데 우리팀이 MVP를 수상하게 되었다.

팀으로써 함께 협력하며 재미있게 또한 열심히 모두 책임감을 가지고 작업하였고, MVP까지 수상하게 되어 너무 뿌듯하고 즐거운 경험이자 추억으로 남게 되었다.
